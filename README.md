# Yandex Tracker Notifier & Dashboard

Chrome-расширение для удобной работы с Yandex Tracker: отслеживание входящих задач, push-уведомления об изменениях статусов, дашборд со счетчиками и встроенный таймер для логирования затраченного времени (worklog).

## Основные возможности

- 🔔 **Уведомления (Push):** Мгновенно узнавайте, когда задача переходит в новый статус (например, из "В работе" в "Ревью").
- 📊 **Матричный Дашборд:** Компактная сводка задач по статусам прямо в попапе. Задачи разбиты по вашим ролям (Исполнитель 👨‍💻, Автор 📝, Наблюдатель 👀). Клик по роли или статусу открывает нужный фильтр в Трекере.
- ⏱️ **Точный таймер учета времени:** Встроенный секундомер с отображением часов, минут и секунд (ЧЧ:ММ:СС). В один клик округляет и отправляет потраченное время в поле "Затраченное время" вашей задачи.
- 💬 **Быстрое цитирование (Quote):** Выделите любой текст в описании задачи или комментарии Я.Трекера, и рядом появится кнопка "Цитата" для мгновенного переноса текста в ваш ответ.
- ⚙️ **Гибкая настройка:** Полная кастомизация того, какие задачи отслеживать с каким интервалом и как округлять время.

## 🚀 Установка

1. Скачайте исходный код расширения в папку на вашем компьютере.
2. Откройте Google Chrome и перейдите по адресу `chrome://extensions/`.
3. В правом верхнем углу включите тумблер **"Режим разработчика"** (Developer mode).
4. Нажмите кнопку **"Загрузить распакованное расширение"** (Load unpacked) и выберите папку с исходным кодом.
5. Расширение появится в браузере. Рекомендуем нажать "Скрепочку" 📌 на панели Chrome, чтобы иконка Трекера всегда была на виду.

## 🔑 Начальная настройка (Как получить токен)

Для работы расширению нужен доступ к вашему Yandex Tracker. Он осуществляется через безопасный [OAuth-токен](https://yandex.ru/support/tracker/ru/concepts/access).

### Шаг 1: Создание OAuth-приложения
1. Перейдите на сервис создания Яндекс OAuth-приложений: [https://oauth.yandex.ru/client/new](https://oauth.yandex.ru/client/new)
2. Заполните базовые поля:
   - **Название приложения:** любое (например, `My Tracker Dashboard`).
   - **Платформы:** выберите **Веб-сервисы** и вставьте URL `https://oauth.yandex.ru/verification_code` в поле "Redirect URI".
3. В разделе **Доступ к данным** (Scopes) найдите блок `Yandex Tracker API` (или введите в поиск слова `tracker`). Вам нужно проставить **две обязательные галочки**:
   - Чтение из трекера (`tracker:read`) — необходимо для показа задач на дашборде и проверки обновлений.
   - Запись в трекер (`tracker:write`) — **Обязательно для работы таймера!** Без этого права расширение не сможет сохранить (залогировать) время в задачу и будет выдавать `Ошибка: API 403`.
4. Нажмите **"Создать приложение"**.

### Шаг 2: Получение токена
1. На странице вашего только что созданного приложения скопируйте **ClientID**.
2. Подставьте этот ClientID в конец следующей ссылки и перейдите по ней в браузере: `<https://oauth.yandex.ru/authorize?response_type=token&client_id=ВАШ_CLIENT_ID>`
3. Нажмите "Войти/Разрешить".
4. Яндекс выдаст вам длинный токен, начинающийся с `y0_AgAAAA...` — **скопируйте его**.

### Шаг 3: Настройка расширения
1. Кликните по иконке расширения в Chrome и нажмите на значок ⚙️ (Шестеренка) в правом верхнем углу.
2. Вставьте ваш токен в поле **"OAuth Token"**.
3. Заполните поле **Org ID**:
   - Его можно найти в ссылке вашего трекера: если ссылка `tracker.yandex.ru/company/12345/`, то ID — `12345`.
   - Если вы используете Yandex 360, укажите тип организации `Яндекс 360`. Если компания на базе Yandex Cloud Organization — выберите `Yandex Cloud`.
4. Нажмите **"Загрузить/Обновить статусы"**. Если всё введено верно, появится зелёная надпись с перечнем загруженных статусов.
5. Прокрутите вниз и нажмите **"Сохранить настройки"**.

## 🛠 Описание настроек

### Системные настройки
- **Язык статусов:** Влияет на то, на каком языке API Яндекса будет возвращать имена статусов (Русский/Английский) в Дашборд.
- **Интервал проверки:** Как часто (в минутах) расширение будет в фоне опрашивать Яндекс на предмет изменений в задачах для отправки Push-уведомлений. Рекомендуется 3-5 минут.

### Таймер задач (Учет времени)
- **Округлять до:** Вы можете выбрать шаг округления (например, 30 минут, 1 час или Пол дня - 4 часа). 
- **Направление округления:**
  - *Математическое:* если вы работали 14 минут при шаге округления "30 минут", отправится 0. Если 16 минут — отправится 30.
  - *В большую сторону:* даже 1 минута работы округлится до выставленного лимита (например, до 30 минут или до 4 часов).
  - *В меньшую сторону:* время всегда будет "срезаться" до ровных отрезков.

### Правила уведомлений
Здесь вы конструируете условия, при которых Chrome покажет вам системное уведомление.
Например:
- Роль: `Исполнитель` / Статус: `Требует информации` -> Вы получите уведомление, если в задачу, где вы исполнитель, перевели в статус "Требует инфо".

### Настройки Дашборда
- **Роли:** Чьи задачи показывать в попапе. Обычно хватает ролей `Assignee` (Мои задачи), `Author` (Где я создатель) и `Followers` (Где я наблюдатель).
- **Статусы:** Отметьте те статусы, которые вас интересуют. Нет смысла выводить в дашборд "Закрытые" задачи. Оставьте только самое важное: "В работе", "Открыт", "На ревью" и т.д.

## ⏱ Как пользоваться таймером
1. Откройте попап расширения.
2. В верхнее поле "Ссылка на задачу..." вставьте либо ключ (например, `WEBDEV-123`), либо просто скопируйте всю вкладку (ссылку) из браузера `https://tracker.yandex.ru/WEBDEV-123` — скрипт сам достанет нужный ключ.
3. Как только вы уберете курсор с поля, скрипт свяжется с Яндексом и подтянет ту сумму времени, которую вы **уже вложили** в эту задачу ранее. Она добавится к текущему времени на таймере.
4. Нажмите ▶️ (Play). Из-под капота запустится посекундный отсчет. Вы можете закрыть попап — время будет идти в памяти браузера.
5. По завершении этапа работы, откройте попап и нажмите ⏹ (Stop) или паузу ⏸. 
6. Расширение автоматически выведет разницу набежавшего времени, округлит её по вашим настройкам (от 1 минуты до 8 часов) и отправит в Tracker как Worklog (Затраченное время).

> **Важно:** при отправке данных (нажатии кнопки "Сохранить") таймер шлёт именно *добавленное текущее время* как новую сессию. Это защищает вас от задвоения всего времени.
